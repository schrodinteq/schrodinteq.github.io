古いバージョンのqiskitインストールについて

任意のバージョンのqiskitを使いたいと思ったことはありませんか？
例えばこの記事を書いてる時点での最新のqiskitバージョンは1.2.4ですが、このバージョンでは「IBM Certified Associate Developer - Quantum Computation using Qiskit v0.2X」の試験で出題されるBasicAerが利用できません。
そこで試験に対応したqiskitバージョンの0.29.1をインストールしようとしたのですが、少し手こずったので記事にします。
この手順の前提としてインストール先の環境はqiskit関連のパッケージが何も入っていない状態を想定しています。
まずは、pipでインストールできるqiskitのバージョンは以下で確認できます。

pip index versions qiskit

では、下記のようにただ、qiskitのバージョンを指定してインストールした場合についてです。

pip install qiskit==0.29.1

この後に適当な下記コードを実行します。

from qiskit import QuantumCircuit

qc = QuantumCircuit(2, 2)
qc.h(0)
qc.cx(0,1)
qc.draw("mpl")

するとこのようなエラーが出ます。

MissingOptionalLibraryError: "The 'Matplotlib' library is required to use 'MatplotlibDrawer'. You can install it with 'pip install matplotlib'."

この後にmatplotlibをインストールして、再実行してもバージョンが合わずにまた失敗してしまいます。

ではどうすれば良いのかというと、下記のように一気に必要となるパッケージをインストールしてしまうことでうまく動きました。

コードも問題なく実行できてます。
